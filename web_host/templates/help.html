NDVI Analytics System

Help

This page contains details on using the system and additional useful information.

How To

To analyse your image follow these steps:

  1. Record data
  2. Transfer images to the file location "static/images"
  3. In the navigation pane click Analysis
  4. Use the dropdown box to select the image you wish to analyse
  5. Click "Go"
  6. View results
To compare two images:
  1. Record data
  2. Transfer images to the file location "static/images"
  3. In the navigation pane click Image Match
  4. Use the dropdown box to select the two images you wish to compare
  5. Click "Go"
  6. View results
Note: Next to the "Image Match" subtitle will be the result of the match. If "True" the images are of the same location else if "False" the images are of different locations.

To use the fullsize images go to the file location stated with the tag "@location"

Interpreting the Results of Analysis

Pre Proccessed Image

This image is the result of pre-processing the original image. In this implementation no pre-processing has been completed but could be an important aspect for some users and therefore the foundations for adaptation were developed.

NDVI Grey

This image is the result of applying the NDVI equation to the original image. Darker pixels indicate lower NDVI whilst lighter pixels indicate higher NDVI.

NDVI Colour

This image is the result of applying the NDVI equation to the original image but preserving the colour channels. Red(er) pixels indicate lower NDVI whilst greener pixels indicate higher NDVI.

NDVI Colour Bar

This image was created to allow teh user to quantify the NDVI of spefic regions. The values on the bar correspond to dead/inanimate, unhealthy, healthy and very healthy values of NDVI (higher is healthier).

NDVI Colour Map

This uses the NDVI Grey image and applies thresholding to colourmap the crop regions. The areas are labled but the shape can sometimes hide the labels. Black is inanimate, red is unhealthy, yellow is healthy and green is very healthy.

Object detection

This uses the NDVI Grey Image and uses blob detection to recognise object in the image. The largest blob is taken as the main crop and has an edge effect value assigned to it where lower scores are better. Anomolies are then labled and circled and are likely weeds or other intrusive objects.

Crop Extraction

This uses the NDVI Grey Image and uses blob detection to recognise object in the image. The largest blob is taken as the main crop and is segmented and cropped. This provide the minimum image that contains the main crop and can be used for further analysis with machine learning and other techniques.
